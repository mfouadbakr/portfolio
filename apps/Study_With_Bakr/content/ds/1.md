# 1. Big O Complexity

Big O notation describes how fast an algorithm grows in terms of time or space as input size increases.

---

## Why Big O Matters

It helps us compare algorithms and choose the most efficient one.

---

## Common Complexities

### O(1) — Constant Time
Execution time does not change with input size.

    int x = 10;
    cout << x;

---

### O(n) — Linear Time
Time increases with input size.

    for (int i = 0; i < n; i++) {
        cout << i;
    }

---

### O(n²) — Quadratic Time
Nested loops.

    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            cout << i << j;
        }
    }

---

### O(log n) — Logarithmic Time
Used in binary search.

    while (n > 1) {
        n = n / 2;
    }

---

### O(n log n)
Used in efficient sorting algorithms like merge sort.

---

## Space Complexity

Measures memory usage of an algorithm.

---

## Example Comparison

| Algorithm | Complexity |
|----------|------------|
| Linear Search | O(n) |
| Binary Search | O(log n) |
| Bubble Sort | O(n²) |

---

## Best vs Worst Case

- Best case → fastest scenario  
- Worst case → slowest scenario  
- Average case → typical performance  

---

## Example: Linear Search

    int search(int arr[], int n, int key) {
        for (int i = 0; i < n; i++) {
            if (arr[i] == key)
                return i;
        }
        return -1;
    }

---

## Summary
- Big O measures efficiency
- Lower is better
- Important for comparing algorithms
