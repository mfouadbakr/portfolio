# 7. Hash Tables

A hash table is a data structure used to store key-value pairs and provides very fast access using a hash function.

---

# WHAT IS HASHING?

Hashing converts a key into an index using a hash function.

Example:
    key → hash function → index

---

# WHY HASH TABLES?

- Fast search (O(1) average case)
- Efficient insertion and deletion
- Used in databases and caching

---

# HASH FUNCTION

A simple hash function:

    index = key % size

---

# COLLISION

A collision happens when two keys map to the same index.

---

## Collision Handling Methods

### 1. Chaining
Store multiple values in the same index using a list.

---

### 2. Open Addressing
Find another empty slot.

---

# IMPLEMENTATION (SIMPLE HASH TABLE)

    const int SIZE = 10;
    int table[SIZE];

---

## Initialize Table

    for (int i = 0; i < SIZE; i++) {
        table[i] = -1;
    }

---

## Insert Function

    void insert(int key) {
        int index = key % SIZE;

        while (table[index] != -1) {
            index = (index + 1) % SIZE;
        }

        table[index] = key;
    }

---

## Search Function

    bool search(int key) {
        int index = key % SIZE;

        int start = index;

        while (table[index] != -1) {
            if (table[index] == key)
                return true;

            index = (index + 1) % SIZE;

            if (index == start)
                break;
        }

        return false;
    }

---

## Delete Function

    void remove(int key) {
        int index = key % SIZE;

        while (table[index] != -1) {
            if (table[index] == key) {
                table[index] = -2; // deleted marker
                return;
            }
            index = (index + 1) % SIZE;
        }
    }

---

# ADVANTAGES

- Very fast lookup
- Efficient for large datasets
- Used in real-world systems

---

# DISADVANTAGES

- Collisions
- Poor hash function reduces performance
- Memory overhead

---

# APPLICATIONS

- Database indexing
- Caching
- Compilers (symbol tables)
- Password storage

---

# SUMMARY

- Hash table = key-value storage
- Uses hash function to map keys
- Collisions must be handled
- Average O(1) performance
