# 8. Evaluation Metrics

Evaluation metrics are used to measure how well a machine learning model performs.

---

## Why Evaluation Matters

A model is useless if we cannot measure its accuracy and performance.

---

# Classification Metrics

## 1. Accuracy

:contentReference[oaicite:0]{index=0}

Measures how many predictions were correct.

---

## 2. Precision

:contentReference[oaicite:1]{index=1}

Measures how many predicted positives were actually correct.

---

## 3. Recall

:contentReference[oaicite:2]{index=2}

Measures how many actual positives were correctly identified.

---

## 4. F1 Score

:contentReference[oaicite:3]{index=3}

Balance between precision and recall.

---

## Confusion Matrix

A table used to evaluate classification results:

- TP → True Positive  
- TN → True Negative  
- FP → False Positive  
- FN → False Negative  

---

# Regression Metrics

## 5. Mean Absolute Error (MAE)

:contentReference[oaicite:4]{index=4}

Measures average absolute errors.

---

## 6. Mean Squared Error (MSE)

:contentReference[oaicite:5]{index=5}

Penalizes large errors more.

---

## 7. Root Mean Squared Error (RMSE)

:contentReference[oaicite:6]{index=6}

Square root of MSE for interpretability.

---

## Python Example

    from sklearn.metrics import accuracy_score, precision_score, recall_score

    y_true = [1, 0, 1, 1]
    y_pred = [1, 0, 0, 1]

    print(accuracy_score(y_true, y_pred))
    print(precision_score(y_true, y_pred))
    print(recall_score(y_true, y_pred))

---

## Overfitting vs Underfitting

- Overfitting → model learns too much noise  
- Underfitting → model is too simple  

---

## Cross Validation

Technique to test model on different data splits to ensure reliability.

---

## Summary
- Accuracy measures correctness
- Precision measures exactness
- Recall measures completeness
- F1 balances precision and recall
- MSE/MAE measure regression error
