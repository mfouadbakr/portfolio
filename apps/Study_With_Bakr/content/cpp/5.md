# 5. Pointers & Memory in C++

Pointers are variables that store memory addresses. They are powerful but require careful handling.

---

## What is a Pointer?

A pointer stores the address of another variable.

    int x = 10;
    int* ptr = &x;

---

## Pointer Basics

- `&` → address-of operator  
- `*` → dereference operator  

    int x = 5;
    int* p = &x;

    cout << p;   // address of x
    cout << *p;  // value of x (5)

---

## Changing Value Using Pointer

    int x = 10;
    int* p = &x;

    *p = 20;

    cout << x; // 20

---

## Pointer and Arrays

Arrays and pointers are closely related.

    int arr[3] = {10, 20, 30};

    cout << arr[0];     // 10
    cout << *(arr + 1); // 20

---

## Pointer Arithmetic

    int arr[3] = {1, 2, 3};
    int* p = arr;

    cout << *p;       // 1
    cout << *(p + 1); // 2
    cout << *(p + 2); // 3

---

## Null Pointer

A pointer that points to nothing.

    int* p = nullptr;

---

## Dynamic Memory Allocation

Using `new` and `delete`.

### Allocate memory:

    int* p = new int;
    *p = 50;

    cout << *p;

### Free memory:

    delete p;

---

## Dynamic Arrays

    int n = 5;
    int* arr = new int[n];

    for (int i = 0; i < n; i++) {
        arr[i] = i + 1;
    }

    delete[] arr;

---

## Dangling Pointer

A pointer pointing to deleted memory.

    int* p = new int(10);
    delete p;
    // p is now a dangling pointer

---

## Memory Leak

Happens when memory is allocated but not freed.

    int* p = new int(5);
    // no delete → memory leak

---

## Summary
- Pointers store addresses
- Use `*` to access value
- Use `new` and `delete` for dynamic memory
- Always free memory to avoid leaks
