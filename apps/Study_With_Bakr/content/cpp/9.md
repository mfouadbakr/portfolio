# 9. Try-Catch & Exceptions in C++

Exceptions are used to handle runtime errors and prevent program crashes.

---

## What is an Exception?

An exception is an error that occurs during program execution.

---

## Try-Catch Block

Used to handle exceptions safely.

    try {
        int a = 10, b = 0;
        if (b == 0)
            throw "Division by zero error";

        cout << a / b;
    }
    catch (const char* msg) {
        cout << msg;
    }

---

## Multiple Catch Blocks

    try {
        int x = -1;

        if (x < 0)
            throw x;
    }
    catch (int e) {
        cout << "Negative number: " << e;
    }
    catch (...) {
        cout << "Unknown exception";
    }

---

## Standard Exceptions

C++ provides built-in exception classes.

    #include <stdexcept>

    try {
        throw runtime_error("Something went wrong");
    }
    catch (runtime_error& e) {
        cout << e.what();
    }

---

## Using throw

Used to signal an exception.

    void checkAge(int age) {
        if (age < 18)
            throw "Not allowed";
    }

---

## Exception Handling Flow

1. Code inside `try` runs
2. If error occurs → `throw`
3. Control moves to `catch`
4. Error is handled

---

## Nested Try-Catch

    try {
        try {
            throw 10;
        }
        catch (int x) {
            cout << "Handled inside: " << x << endl;
            throw;
        }
    }
    catch (int x) {
        cout << "Handled outside: " << x;
    }

---

## Best Practices

- Use exceptions for unexpected errors
- Avoid overusing exceptions
- Catch specific exceptions first
- Clean up resources properly

---

## Summary
- Exceptions handle runtime errors
- Use `try`, `throw`, and `catch`
- Multiple catch blocks allowed
- Standard exceptions are available
